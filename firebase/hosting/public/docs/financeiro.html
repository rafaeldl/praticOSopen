Visão Geral do Sistema Financeiro

O PráticOS oferece um sistema financeiro completo e integrado para controle de pagamentos, descontos e acompanhamento de faturamento. Todas as transações são registradas com histórico detalhado e vinculadas às Ordens de Serviço.

Integração com RBAC

O acesso aos dados financeiros é controlado por perfil. Técnicos e Supervisores não visualizam valores, garantindo segurança das informações.

Pagamentos Parciais

Registre múltiplos pagamentos com histórico completo de transações

Descontos

Aplique descontos com descrição e rastreabilidade completa

Dashboard

Acompanhe faturamento e valores a receber em tempo real

Status de Pagamento

O sistema utiliza três status para indicar a situação financeira de cada Ordem de Serviço:

A Receber

Nenhum pagamento registrado. O valor total da OS está pendente.

Parcial

Pagamento parcial registrado. Ainda há saldo a receber do cliente.

Pago

Totalmente pago. Não há saldo pendente para esta OS.

Nota Técnica

O status "Parcial" é calculado automaticamente com base no valor pago. No banco de dados, apenas "Pago" e "A Receber" são armazenados para manter compatibilidade com filtros existentes.

Gestão de Pagamentos

A tela de pagamentos centraliza todas as operações financeiras de uma OS, permitindo registrar pagamentos, aplicar descontos e visualizar o histórico completo.

Como Acessar

  1. Abra uma Ordem de Serviço existente
  2. Na seção Resumo, clique em Total ou no status de Pagamento
  3. A tela de gestão de pagamentos será aberta automaticamente

Funcionalidades Disponíveis

Resumo Financeiro

  • Total da OS
  • Descontos aplicados
  • Valor já pago
  • Saldo restante

Registrar Transação

  • Alternar entre Pagamento e Desconto
  • Valor pré-preenchido com saldo
  • Campo de observação opcional
  • Ação rápida "Pagar total"

Histórico

  • Lista de pagamentos e descontos
  • Ordenado por data
  • Deslize para remover
  • Descrição e responsável

Regras de Negócio

O sistema aplica validações para garantir a integridade dos dados financeiros e o fluxo correto de pagamentos.

Pagamentos por Status da OS

A possibilidade de registrar pagamentos depende do status atual da Ordem de Serviço:

Status da OS Pode Registrar Pagamento? Auto-Aprovação
Orçamento ✓ Sim ✓ Transita para Aprovado
Aprovado ✓ Sim ✗ Não
Em Andamento ✓ Sim ✗ Não
Concluído ✓ Sim ✗ Não
Cancelado ✗ Não ✗ Não

Auto-Aprovação de Orçamentos

Ao registrar um pagamento ou desconto em uma OS com status Orçamento, ela é automaticamente aprovada e transita para o status Aprovado. Isso elimina a necessidade de aprovação manual antes de registrar o primeiro pagamento.

Validações Aplicadas

  • Valor mínimo: O valor do pagamento ou desconto deve ser maior que zero
  • Valor máximo: O valor não pode exceder o saldo restante da OS
  • Status válido: A OS não pode estar cancelada (orçamentos são permitidos e auto-aprovados)

Dashboard Financeiro

O dashboard oferece uma visão consolidada do faturamento da empresa, com indicadores em tempo real e ranking de clientes.

Como Acessar

Acesse através do Menu lateral → Dashboard. Requer a permissão viewFinancialReports (disponível para Admin e Gerente).

Indicadores Disponíveis

Faturamento Total

Soma dos valores totais de todas as OS no período selecionado

Valor Recebido

Soma de todos os pagamentos registrados, incluindo parciais

A Receber

Saldo pendente das OS não totalmente pagas

OS Pagas

Quantidade de OS com pagamento completo

Ranking de Clientes

O dashboard exibe os clientes ordenados por valor, permitindo identificar:

  • Maiores clientes: Ordenados pelo valor total de OS
  • Maiores devedores: Ordenados pelo valor a receber pendente

Impressão (PDF)

O PDF da Ordem de Serviço inclui o resumo financeiro completo, adaptando-se automaticamente ao status de pagamento.

Informações no PDF

Resumo de Totais

  • Serviços (quando houver)
  • Produtos (quando houver)
  • Subtotal
  • Desconto (quando aplicado)
  • Total
  • Já pago (em pagamentos parciais)

Rodapé do Resumo

O rodapé exibe o status financeiro com cores diferenciadas:

Proteção de Dados Financeiros

Para usuários sem permissão viewPrices, os seguintes elementos ficam ocultos: valores da OS, status de pagamento, preços de serviços/produtos, geração de PDF e filtros de pagamento.

Perguntas Frequentes

Posso registrar pagamento em um orçamento?

Não. Pagamentos só são liberados a partir do status "Aprovado". Primeiro aprove o orçamento, depois registre o pagamento.

Como aplicar um desconto?

Na tela de pagamentos, alterne para "Desconto" no seletor superior, informe o valor e adicione uma descrição (ex: "Desconto fidelidade").

Posso remover um pagamento registrado?

Sim. No histórico de transações, deslize o item para a esquerda e confirme a remoção. A ação é registrada no histórico.

O que significa status "Parcial"?

Indica que existem pagamentos registrados, mas o valor total ainda não foi quitado. O saldo restante é exibido no resumo.

Como marcar uma OS como totalmente paga?

Use a ação rápida "Pagar valor total" na tela de pagamentos, ou registre um pagamento com o valor exato do saldo restante.

Os pagamentos parciais aparecem no PDF?

Sim. O PDF exibe "Já pago" com o valor recebido e "SALDO RESTANTE" destacado em laranja no rodapé.

Técnicos podem ver valores financeiros?

Não. Técnicos e Supervisores não têm acesso a dados financeiros. Apenas Admin, Gerente e Consultor (próprias OS) visualizam valores.

O dashboard considera pagamentos parciais?

Sim. O indicador "Valor Recebido" soma todos os pagamentos registrados, e "A Receber" exibe apenas o saldo pendente.
